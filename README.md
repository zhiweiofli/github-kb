# GitHub Knowledge Base (github-kb)

A filesystem-first approach to managing your personal GitHub knowledge base.

## What is this?

github-kb turns GitHub into a **searchable, ownable local knowledge system**.

Instead of relying on browser history or remote bookmarks, relevant GitHub projects are cloned into a fixed directory, summarized, and indexed in a single catalog file.

## Features

- **Search GitHub** via `gh` CLI (repos, issues, PRs)
- **Clone repositories** into organized local storage
- **Retrieve information** using local file tools
- **Maintain a catalog** as human-readable index

## Quick Start

### Prerequisites

```bash
# Install GitHub CLI
brew install gh

# Authenticate
gh auth login
```

### Usage

Use natural language or slash commands:

```
# Search for repositories
/search-github chatbot framework python

# Clone a repository
/clone-repo anthropics/claude-code

# Update the catalog
/update-catalog
```

## Philosophy

**Low complexity, high control.**

This skill favors:
- Filesystem ownership over cloud dependency
- Explicit structure over magic discovery
- Human-curated summaries over auto-generated content
- CLI-based search over complex APIs

The result is a durable, portable, and inspectable personal GitHub knowledge system.

## Non-Goals

- No mass crawling or scraping
- No vector databases or embeddings
- No private repo access without explicit auth
- No automatic summarization without reading source
